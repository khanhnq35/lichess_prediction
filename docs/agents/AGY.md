# agy Handoff

## Nhiệm Vụ Phù Hợp

- Đọc `reports/working/` và `artifacts/` để tổng hợp narrative.
- So sánh lightweight production với heavy/Stockfish experiments.
- Kiểm tra README/final report có giải thích leakage prevention, reproducibility, và LLM usage rõ chưa.
- Đề xuất experiment tiếp nhưng ghi rõ dependency, runtime, và rủi ro submission size.

## File Nên Đọc Trước

1. `problem.md`
2. `docs/PROJECT_MAP.md`
3. `docs/PROBLEM_TRACEABILITY.md`
4. `docs/ARTIFACT_INDEX.md`
5. `docs/DECISION_LOG.md`

## Cẩn Trọng

- Một số report cũ trong `reports/working/` vẫn nhắc path dạng `outputs_*` trước khi dọn. Artifact tương ứng hiện nằm trong `artifacts/`.
- Kết quả Stockfish/heavy rất mạnh nhưng không tự động hợp lệ cho final lightweight submission.
